    # Chapter 6. Conclusion and Future Work

    ## 6.1 Conclusion

    In conclusion, the Web-based Event Ticketing System has successfully achieved its initial objectives. By providing a user-friendly and intuitive web interface, the system significantly enhances the efficiency and accessibility of event data management and ticket distribution for all stakeholders involved. Users are able to interact with the system easily, reducing the time and effort required to perform routine tasks such as discovering events, booking tickets, managing event layouts, and conducting attendee check-ins.

    One of the key strengths of the system lies in the implementation of secure authentication and role-based access control, combined with a resilient microservices architecture. These mechanisms ensure that users—whether Customers, Organizers, Staff, or Administrators—can only access features and data that correspond to their assigned roles, thereby improving data security and preventing unauthorized operations. Core system functionalities have been fully implemented and integrated with robust concurrency controls, allowing ticket and order information to be managed in a structured, consistent, and reliable manner without the risk of overselling.

    Overall, the system supports effective handling of high-concurrency transactions and contributes to greater transparency in event administration activities. By centralizing ticket allocation data and providing controlled, real-time access, the Web-based Event Ticketing System offers a solid foundation for supporting the dynamic and demanding operations of the modern event and entertainment industry.

    ## 6.2 Future Work

    Although the system has met its primary goals, there are several potential improvements that can be considered for future development. To enhance interaction and communication between users, an advanced AI-driven recommendation engine and a dynamic pricing model could be introduced, allowing event organizers and customers to exchange value and discover events more conveniently within the system.

    In addition, the system could be extended with advanced functionalities such as a dedicated native mobile application for offline ticket scanning and student/staff self-service portals. These features would reduce manual administrative work at the venue and further improve user convenience. From a technical perspective, performance optimization should also be prioritized to ensure faster response times, especially when the system handles a large number of concurrent users during blockbuster event sales. Continuous improvement in system scalability, usability, and performance would help the platform better meet the evolving needs of event management institutions.
